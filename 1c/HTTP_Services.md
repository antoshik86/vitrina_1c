# HTTP-сервисы витрины (1С:УНФ)

## Публикация

В конфигурации УНФ создать HTTP-сервис `vitrina` (или настроить через Web-сервисы / Обработки).

**Путь:** `/hs/vitrina/...`

Все ответы в формате **JSON**, кодировка **UTF-8**.

---

## 1. GET /hs/vitrina/catalog

Получение списка товаров и категорий.

### Ответ:
```json
{
  "products": [
    {
      "id": "uuid-1",
      "name": "Стул офисный",
      "article": "СТ-001",
      "price": 4500.00,
      "image": "https://example.com/img/chair.jpg",
      "categoryId": "cat-1",
      "category": "Мебель",
      "description": "Описание товара"
    }
  ],
  "categories": [
    { "id": "cat-1", "name": "Мебель" },
    { "id": "cat-2", "name": "Оргтехника" }
  ]
}
```

### 1С-реализация (пример):
```bsl
Функция Каталог(Запрос)
    // Выборка номенклатуры из УНФ (справочник Номенклатура)
    // Учитывать: Товары, не помеченные на удаление, с остатками > 0
    // Для изображений: использовать адреса временных файлов или URL
    Возврат Новый Запрос("
        |ВЫБРАТЬ
        |   Номенклатура.Ссылка КАК Ид,
        |   Номенклатура.Наименование КАК Наименование,
        |   Номенклатура.Артикул КАК Артикул,
        |   Номенклатура.ЦенаПродажи КАК Цена,
        |   Номенклатура.Комментарий КАК Описание,
        |   Номенклатура.ВидНоменклатуры.Наименование КАК Категория
        |ИЗ
        |   Справочник.Номенклатура КАК Номенклатура
        |ГДЕ
        |   Номенклатура.НеУдалять = Ложь
        |   И Номенклатура.ПометкаУдаления = Ложь
        |   И Номенклатура.ЭтоГруппа = Ложь
        |   И Номенклатура.ТипНоменклатуры = ЗНАЧЕНИЕ(Перечисление.ТипыНоменклатуры.Товар)");
КонецФункции
```

## 3. POST /hs/vitrina/order

Создание заказа из корзины.

### Запрос:
```json
{
  "items": [
    { "id": "uuid-1", "name": "Стул офисный", "qty": 2, "price": 4500.00 }
  ],
  "clientInfo": {
    "name": "Иван Иванов",
    "phone": "+79991234567",
    "comment": "Позвонить после 18:00"
  }
}
```

### Ответ:
```json
{
  "success": true,
  "orderNumber": "ЗА-000001",
  "message": "Заказ принят"
}
```

### 1С-реализация:
- Создать документ `ЗаказПокупателя` (УНФ)
- Заполнить табличную часть из items
- Заполнить контактную информацию

## 4. POST /hs/vitrina/chat/ai

Вопрос к AI-ассистенту.

### Запрос:
```json
{
  "text": "Какие стулья подойдут для офиса?"
}
```

### Ответ:
```json
{
  "answer": "Для офиса рекомендуем стулья с регулировкой высоты и поддержкой поясницы. В нашем каталоге есть модели..."
}
```

**Варианты реализации AI:**
- Использовать HTTP-запрос к OpenAI API / GigaChat API из 1С
- Или локально сопоставлять ключевые слова с товарами

## Важные замечания

### CORS
Для работы с фронтенда на стороннем хостинге, в 1С необходимо добавить заголовки CORS:

```bsl
Ответ.Заголовки.Вставить("Access-Control-Allow-Origin", "*");
Ответ.Заголовки.Вставить("Access-Control-Allow-Methods", "GET, POST, OPTIONS");
Ответ.Заголовки.Вставить("Access-Control-Allow-Headers", "Content-Type");
```

### Аутентификация
Для теста — базовая HTTP-аутентификация или по токену. Фронтенд передаёт в заголовке `Authorization: Basic base64(логин:пароль)`.
